Props — это данные, которые передаются от одного компонента к другому.
Очень просто: props = параметры компонента.

Один из любимый примеров указания пропсов 

В JS , я уже создавал классы в которые передавались данные и на основе этого формировались карточки, в реакте так же , у компонентов есть свои свойства 

function whoAmI () {
	return (
		<div>
			<h1>My name is , surname = </h1>
			<a href=""></a>
		</div>
	)
}

Это компонент с данными, которые захаркорджены, это просто заготовка

А для работы с ним , нужно прописывать аргумент, такой как Props , откуда и отдадутся данные 
Пропс это объект со всеми вещами которые мы в него передадим 

function whoAmI (props) {
	return (
		<div>
			<h1>My name is {props.name} , surname - {props.surname}</h1>
			<a href="{props.link}"></a>
		</div>
	)
}

Теперь компонент для того чтобы создаться будет требовать передачи данных 

И теперь я могу вставить этот компонент например в приложение, как я делал с маленькими и потом вставлял в одно большое и указать ему определенные атрибуты 

<WhoAmI name ='John' surname='Smith' link='facebook.com'/>

Иногда компоненты могут быть без пропса, но он в любом случае будет сформирован, увидеть я это смогу в консоли при формировании объекта 

после я могу добавить ещё один компонент , подставив туда другие данные, допустим заменив линк  

<WhoAmI name ='John' surname='Smith' link='vk.com'/>

А так же объект можно деструктурировать 

function whoAmI (name, surname, link) {
	return (
		<div>
			<h1>My name is {props.name} , surname - {props.surname}</h1>
			<a href="{props.link}"></a>
		</div>
	)
}

указав туда атрибуты name, surname, link - теперь для передачи данных в компонент можно не указывать пропс через точку , а уже пользоваться арибутами удобнее name, surname, link 

Важная деталь которую нужно запомнить про пропсы - ( ЗНАЧЕНИЯ АТРИБУТОВ КОТОРЫЕ ПЕРЕДАЮТСЯ УЖЕ В <WhoAmI /> уже изменить нельзя , они только для чтения и если мы хотим их изменить, для того чтобы динамически изменить значение, мы должны полностью перерисовать компонет )
Как работать динамически - я выучу немного позже 

Пропс - мы используем в компоненте, но никогда не записываем ничего внутри него же 
Состовляющие блоки компонентов это элементы и в данном случае заголовок h1 , это элемент и помещенные элементы - изменить нельзя. Чтобы его изменить - нужно целый компонент отрендерить заново. 

Если компонент начнем сильно разростаться нам нужно разделить его на компоненты. 

В пропс можно передать что угодно , даже объект 

<WhoAmI name ='{{firstName}}' surname='Smith' link='vk.com'/>

но если в компонент постараться вставить именно объект от не будет работать , он отдаст ошибку, как мы и видели ранее.

Можно так же передать не какое-то готовое значение , а функцию 

<WhoAmI name ='{}' surname='Smith' link='vk.com'/>

Вставив в name () => {return 'John'}, а потом внутри заголовка 

<h1>My name is {name} , surname - {props.surname}</h1> - заместо name нужно вызвать name() вызвать функцию
